** 방산SW개발 실무(실습) (2026-07-06) **
- 체계공학(System Engineering)의 이해
  * 체계공학이란 사용자 요구사항으로부터 요구사항 분석, 설계, 제작, 검증, 확인, 운용, 폐기에 이르는 모든 단계를 수명주기(Life Cycle) 관점을 고려해
    사용자의 요구사항을 충족하도록 경제적, 균형적으로 체계를 개발하는 방법론
- SE기반 기술검토회의 개요
  * 기술검토회의란 사업관련 모든 이해자(stakeholder)들이 연구개발 진행간 공식긱술검토 시점(SRR~PCA)에서 단계별 설정기준에 맞게 요구사항 분석에 따른
    현재 개발 진척도를 비교/분석해 분야별 완성도를 검토하기 위한 회의
  * 기술검토회의는 SRR에서 CDR까지 하향식 세분화 과정을 통해 구체화 되며 이것은 다시 구현, 통합, 검증 및 확인의 상향식 과정을 거쳐 사용자 요구사항을 충족하는 최종 제품으로 완성
  * 기술검토회의는 해당 단계에 맞는 진척도가 어느 정도 완성수준에 달성한 상태에서 다음 단계로 진입을 위한 사업진행 간 문제점 해결 중심적으로 진행되어야함
- 국방 개발 프로세스 모델
  * V모델을 따름(폭포수 모델의 확장된 형태)
  * 폭포수 모델의 단점을 보완하기 위한 방안 고려 필요(프로토타이핑, Agile 개발 요소 도입 등)
  ** Verification(확인) : Did you build the product right? 명세서대로 제대로 만들었는가?
  ** Validation(검증) : Did you build the right product? 사용자가 원하는 것을 만들었는가?
- 방어선마다 사용하는 무기가 다 다르기 때문에 각각의 과제들로 나오게 됨
- 전 기수 주제
  * On-Device AI 기반 지능형 EO/IR 상황인식 시스템
  * 미래를 여는 CIWS-X 무기체계 개발 (TDD 개발 기반)
- 고객의 만족도를 만들기 위해서 청사진을 미리 제공함
- 기술 검토를 위한 Tailoring과 산출물 비교 현황
  * 효율적 연구개발을 위해서는 산출물 납품에 대해 과제 관리 담당자와 충분한 협의 필요
- 무기체계 연구개발 기술검토회의
  * SRR -> SFR -> PDR -> CDR -> TRR -> FCA -> PCA
- 개발 프로세스 실습
  * '무기체계 소프트웨어 개발 및 관리 매뉴얼"에 기술된 활동
