- :: 스코프 연산자 : 네임스페이스를 사용하여 이름영역 구분
- 지역변수 선언 위치: c++은 사용하기 전에 선언만 하면 됨 + 위치 상관 없음
- 힙 메모리 할당 : malloc(크기) / free() => new 타입 / delete

- c++은 함수 오버로딩가능(Function Overloading)
  * 함수 오버로딩: 동일한 이름의 함수를 여러 개 정의할 수 있음
  * 함수 호출시 컴파일러가 구분할 수 있도록 파라메터의 타입이나 개수를 다르게 해야함
  * 기능은 같으나 입력, 출력 값에 따라서 이름을 다르게 지어야하는 번거로움을 제거
  * 다형성에도 사용될 수 있음
  * const도 오버로딩에 사용할 수 있음
  ** 컴파일러가 인자의 타입을 보고 자동으로 적절한 함수를 찾아감
  // add(int, int)
  // add(int , int, int)
  // add(float, float)
  // add(const char *, const char *)
  // add(1.2f, 3.4f);

- 함수 단점
  * 재귀 호출은 스택 오버플로우 문제 발생위험
  * 함수 호출 -> 함수로 분기 -> 파이프라인 삭제 다시 구축 => 오버헤드 발생
  * 짧은 함수는 매크로나 인라인함수로 대체 => 2~3줄 정도의 짧고, 반복문 없이 처리가 비교적 간단한 함수에 사용
  ** 매크로: 전처리기가 처리. 함수 호출 부분 코드를 교체 -> 사이드 이펙트 발생. 타입 체크 어려움
  ** 인라인: 컴파일러가 처리 타입 체크 가능.

- & -> 참조변수 : 변수의 별칭. 선언시 꼭 초기화 해야함.
  * call by value: 변수 값 복사해서 전달(값, 주소값)
  * call by reference: 참조 변수를 통한 전달(파라미터가 참조변수) -> 가장 효율적이고 좋으니 이렇게 하는 걸 추천
  ** 참조변수는 선언과 동시에 초기화해야하고, nullptr 안 됨. 일반 변수와 똑같이 사용 가능

- C++에서 데이터를 다룬다는 것은 컴퓨터의 RAM(메모리)을 예약하고, 그곳에 값을 쓰고 읽는 행위
  * 변수(Variables) : 데이터를 담는 변하는 상자. 데이터를 저장하기 위해 이름을 붙인 메모리 공간
  * 자료형(Data Types) : 메모리의 크기와 종류 결정. 변수라는 상자의 크기(Byte)와 종류(정수, 실수, 문자 등)를 결정
  * 상수(Constants) : 절대 변하지 않는 값. 프로그램 실행 도중 값이 절대 변하지 않는 데이터 -> 유지보수성 향상, 버그 예방
  ** 리터럴 상수(Literal Constant): 코드에 글자 그대로 적힌 수치값 자체 (예: 100, 3.14, 'A')
  ** 심볼릭 상수(Symbolic Constant): 변수처럼 이름을 가졌지만 값을 바꿀 수 없도록 지정된 상수로, const 키워드 사용

- 객체지향에서 중요한 건 캡슐화 -> class 사용
  * 헤더 파일에 어떻게 쓰겠다고 정의해줘야 함
  * public(어디서든 참조 가능함) 생략하면 기본적으로 private(클래스 안에서만 사용하고 밖에서는 할 수 없게 함)임
  ** 멤버 변수는 private으로 해줌 -> 밖에서 직접적으로 다룰 수 없도록 하기 위함
  * 구조체 멤버 변수와 비슷함

[클래스(Class)와 객체(Object)]
- 객체지향 특징 : 캡슐화, 상속, 다형성(!중요!), 추상화
- 클래스(Class) : 객체를 만들기 위한 '설계도' 혹은 '틀'. 실체는 없고 "이 객체는 어떤 데이터와 기능을 가진다"라고 정의만 해둔 상태
  * 구조체는 변수들만 묶을 수 있었지만, 클래스는 데이터(변수)와 기능(함수)을 한곳에 통째로 묶어 관리
  ** 멤버 변수(Member Variable) : 객체의 '상태'나 '속성'을 나타내는 변수
  ** 멤버 함수(Member Function) : 객체의 '행동'이나 '기능'을 나타내는 함수 -> 메소드
- 객체(Object) / 인스턴스(Instance) : 설계도(클래스)를 바탕으로 컴퓨터 메모리(RAM)에 실제로 구현된 실체
  * 클래스는 하나만 정의해 두면 객체는 메모리가 허용하는 한 무한히 만들어낼 수 있음
- 접근 제어 지시자(Access Specifiers)
  * 가장 넓은 접근제어자는 public : 클래스 외부 어디에서나 자유롭게 접근하여 사용
  * private : 클래스 내부에서만 접근할 수 있고, 외부(main 함수 등)에서는 절대 직접 접근할 수 없음. 보통 중요한 멤버 변수들을 여기에 숨김
  * protected : 상속관계 클래스에만 보이고 다른 클래스에서는 안보임

[생성자(Constructor)와 ~소멸자(Destructor)]
- 생성자는 객체를 생성할 때 자동으로 딱 한 번 실행되며, 이름이 클래스의 이름과 완전히 똑같아야하며, 봔환 타입이 아예 없음
- ~ 소멸자는 객체의 수명이 다해 메모리에서 사라질 때 자동으로 호출되는 마지막 정리 함수
